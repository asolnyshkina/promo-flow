# promo-flow

Интерактивный прототип переходов между экранами в мобильном приложении.
Клиентская работа, поэтому содержимое закрыто паролем.

**https://asolnyshkina.github.io/promo-flow/** — пароль по запросу.

## Как это устроено

В репозитории лежит только `index.html`. Внутри него — форма ввода пароля и
зашифрованный блок: страница целиком (разметка, стили, скрипты и картинки)
упакована в один документ и зашифрована **AES-256-GCM**, ключ выводится из
пароля через **PBKDF2-SHA256, 250 000 итераций** со случайной солью.
Расшифровка происходит в браузере через Web Crypto.

Без пароля из репозитория не достать ничего: открытого текста здесь нет.
Неверный пароль не проходит проверку подлинности GCM — подобрать «почти
верный» нельзя. Стойкость упирается только в сам пароль, поэтому он
случайный, а не придуманный.

Исходники (`src/`) в репозиторий не коммитятся — иначе шифрование не имело
бы смысла.

## Пересборка

```bash
node build.js "пароль"
```

Скрипт собирает `src/` в один самодостаточный HTML, шифрует его и кладёт
результат в `index.html`. Смена пароля — просто пересборка с новым.
Дальше обычный `git push`, GitHub Pages подхватит через минуту-две.

Сборка падает, если после инлайна пропал любой из контрольных маркеров или
осталась ссылка на локальный файл, — молча выкатить сломанную страницу не
даст.

Требуется Node (проверено на 24.x). Открывать надо по `https://` или с
`localhost`: на `file://` браузер не даёт доступ к Web Crypto.
